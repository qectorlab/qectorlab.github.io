Validated scope

What QECTOR supports today.

  • Public package distributed as qector-decoder-v3 on PyPI
  • Python API exposes UnionFindDecoder, BlossomDecoder, and additional decoder workflows
  • Exact-MWPM LER parity with PyMatching on tested rotated-surface-code Stim workloads through d=15
  • Belief-matching is an accuracy mode with lower observed LER on selected correlated workloads, but it is slower than plain MWPM
  • BP-OSD / LDPC / qLDPC workflows are supported and cross-checked on selected workloads
  • Batch paths are tested for CPU bit-identity on supported configurations
  • DOI-backed provenance record: 10.5281/zenodo.20825980
Decoder stack

One library for validated QEC decoding workflows.

Union-Find / FastUF

Fast approximate matching-graph decoding path.

Blossom MWPM

Exact weighted MWPM path used for tested PyMatching LER parity.

Sparse Blossom

Region-growing near-optimal Blossom path, not exact MWPM.

BeliefMatching

Accuracy mode for selected correlated workloads; slower than plain MWPM.

BP-OSD / LDPC

BP-OSD workflows for LDPC and qLDPC experiments.

Streaming

Sliding-window and accumulated streaming decoder workflows.

Claim boundaries

Sales-safe positioning.

  • PyMatching remains the latency leader for exact MWPM on common surface-code workloads
  • Belief-matching is slower because it rebuilds weights per shot
  • Sparse Blossom is near-optimal, not exact MWPM
  • Hardware-specific throughput claims must be regenerated locally before quoting sales or scientific speed claims

Correct positioning

QECTOR's strengths are reproducibility, evidence packaging, accuracy-mode experimentation, BP-OSD / LDPC support, batch workflows, and commercial integration.
