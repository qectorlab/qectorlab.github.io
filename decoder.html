Decoder stack

One library for validated QEC decoding workflows.

Union-Find / FastUF

Fast approximate matching-graph decoding path.

Blossom MWPM

Exact weighted MWPM path used for tested PyMatching LER parity.

Sparse Blossom

Region-growing near-optimal Blossom path, not exact MWPM.

BeliefMatching

Accuracy mode for selected correlated workloads; slower than plain MWPM.

BP-OSD / LDPC

BP-OSD workflows for LDPC and qLDPC experiments.

Streaming

Sliding-window and accumulated streaming decoder workflows.

CPU/GPU batch

CPU, CUDA, and OpenCL batch workflows with CPU bit-identity tests.

AutoDecoder

Calibrated backend selection for CPU/GPU batch workflows.

Honest limitations

Do not overclaim.

  • PyMatching remains the latency leader for exact MWPM on common surface-code workloads
  • Belief-matching is slower because it rebuilds weights per shot
  • Sparse Blossom is near-optimal, not exact MWPM
  • GPU throughput claims must be regenerated on local hardware before quoting sales or scientific speed claims
  • The REST/API layer is local/demo infrastructure unless covered by a separate commercial agreement

Correct positioning

QECTOR's strengths are reproducibility, evidence packaging, source access, accuracy-mode experimentation, BP-OSD / LDPC support, GPU batch workflows, and commercial integration.
