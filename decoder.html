QECTOR Decoder 3.

A Python and Rust quantum error correction decoder package for validation, reproducible artifacts, BP-OSD and LDPC workflows, and commercial evaluation.

Validated scope

What QECTOR supports today.

  • Public package distributed as qector-decoder-v3 on PyPI
  • Python API exposes UnionFindDecoder, BlossomDecoder, and additional decoder workflows
  • PyMatching-compatible validation paths for matching workflows
  • Belief matching for selected correlated workloads
  • BP-OSD, LDPC, and qLDPC workflows for research studies
  • CPU and GPU batch decoding workflows
  • DOI backed provenance record: 10.5281/zenodo.20825980
Decoder stack

Belief Matching

Accuracy mode for selected correlated workloads.

BP-OSD / LDPC

BP-OSD workflows for LDPC and qLDPC experiments.

Streaming

Sliding window and accumulated streaming decoder workflows.

Claim boundaries

Clear technical positioning.

  • PyMatching remains an important reference baseline for exact MWPM workloads
  • Belief matching is an accuracy mode, not a universal speed claim
  • Sparse Blossom is near optimal, not exact MWPM
  • Hardware specific throughput claims should be regenerated locally before quotation
