What's New in v0.5.4

Improved BatchDecoder API

Added BatchDecoder.decode() for single-shot decoding and better support for large batch workflows.

Enhanced BeliefMatching

BeliefMatching now accepts raw NumPy parity-check matrices directly, improving ease of use with custom codes.

Sinter Integration Improvements

QectorSinterDecoder now supports DEM caching for significantly faster repeated Monte-Carlo runs.

Performance & Stability

Multiple internal optimizations and bug fixes for GPU backends (CUDA/OpenCL) and Union-Find decoder stability.

Documentation & Examples

Updated documentation and added more practical examples for Stim + Sinter workflows.

Sinter Integration Benefits

QECTOR provides native support for Sinter, the standard Monte Carlo sampling framework used in the quantum error correction community.

Standardized Benchmarking

Run QECTOR decoders using the same experiment harness as PyMatching. Results are directly comparable.

Reproducible Experiments

Sinter + QECTOR artifact hashing makes benchmarks highly reproducible and auditable.

Easy Head-to-Head Comparisons

Quickly compare QECTOR against PyMatching on identical circuits and noise models.

Performance on Repeated Runs (v0.5.4)

The QectorSinterDecoder now supports DEM caching, significantly speeding up large parameter sweeps.
