Decoder Modes

Decoder paths and correct use

Blossom / Weighted MWPM

LER parity on tested circuit-level workloads

Use when logical accuracy matters and PyMatching-compatible comparison is required. Exact on small/medium validated instances; adaptive-k near-exact at larger distances with LER parity preserved in tested d=13/d=15 audits.

Belief-Matching

accuracy mode

Uses BP on the hyperedge detector graph to reweight matching. Lower observed LER on selected correlated rotated-surface workloads; slower than plain MWPM because matching is rebuilt per shot.

BP-OSD / LDPC

qLDPC-capable

For codes whose mechanisms are not graphlike matching problems, including bivariate-bicycle and other LDPC/qLDPC experiments. Competitive with the ldpc package in tested workloads.

Union-Find / FastUF

fast approximate path

Useful for fast approximate decoding on real QEC matching graphs. Not exact on arbitrary adversarial graphs. Use Blossom for strongest faithfulness guarantees.

CUDA / OpenCL Batch

bit-identical on tested batches

Best for large batches where kernel launch overhead is amortized. GPU speedups depend on batch size, hardware, drivers, and backend availability.

Streaming / Sliding Window

multi-round workflow support

For maintaining state across rounds and future Workbench workflows. Validate against your own circuit, distance, and timing constraints.

Install

Source checkout installation

Public binary availability may vary by platform. For reproducibility and local validation, build from source with maturin.
git clone https://github.com/qectorlab/qector-decoder.git
cd qector-decoder

pip install maturin
maturin develop --release --no-default-features
pip install -e ".[dev]"
pytest

Optional GPU builds

# CUDA
maturin develop --release --no-default-features --features cuda

# OpenCL
maturin develop --release --no-default-features --features opencl
Python Quick Start

Minimal decoder example

import numpy as np
from qector_decoder_v3 import UnionFindDecoder

check_to_qubits = [[0, 1], [1, 2], [2, 3]]
decoder = UnionFindDecoder(check_to_qubits)

syndrome = np.array([0, 1, 0], dtype=np.uint8)
correction = decoder.decode(syndrome)
print(correction)
Honest Limits

What QECTOR does not claim

Not a universal fastest-decoder claim

PyMatching remains the latency leader for exact MWPM on larger tested circuit-level workloads. QECTOR emphasizes reproducibility, multiple workflows, and commercial evidence packaging.

Not fully open-source

QECTOR is source-available for permitted non-commercial use. Commercial use requires a paid license under QECTOR Decoder Source-Available License v1.0.
