Install, test, reproduce, evaluate.

QECTOR Decoder 3 is a source-available Rust/Python quantum error correction decoding platform. This page mirrors the public README so the website is the single readable source for install commands, validation claims, evidence boundaries, and commercial next steps.

Recommended public install

Use the PyPI wheel first.

pip install qector-decoder-v3
python -c "from qector_decoder_v3 import UnionFindDecoder, BlossomDecoder; print('QECTOR OK')"

Use the same Python environment for installation and verification.

Check environment

Confirm Python and pip.

python --version
pip --version
py -0p

On Windows, avoid accidentally using free-threaded Python such as python3.13t.exe. QECTOR v0.5.x publishes standard CPython wheels, not cp313t wheels.

Quick start

Minimal decoder example.

import numpy as np
from qector_decoder_v3 import UnionFindDecoder, BlossomDecoder

check_to_qubits = [[0, 1], [1, 2], [2, 3], [3, 4]]
n_qubits = 5
syndrome = np.array([0, 1, 0, 0], dtype=np.uint8)

uf = UnionFindDecoder(check_to_qubits, n_qubits)
print(uf.decode(syndrome))

mwpm = BlossomDecoder(check_to_qubits, n_qubits)
print(mwpm.decode(syndrome))

Honest limitations.
Union-Find is roughly 3× less accurate than MWPM in the validated workloads. Single-round code-capacity surface-code noise does not produce threshold curves. SparseBlossom batch may return different but valid corrections on degenerate cases. GPU wins only at sufficient batch sizes, typically 4096 or more. CUDA raises a clean RuntimeError when no driver is present.

Correct positioning

QECTOR complements PyMatching, Stim, and Sinter. Do not present it as a universal speed leader, production SaaS, OEM-ready embedded stack, or real-time hardware QEC infrastructure without a specific artifact proving that exact claim.
