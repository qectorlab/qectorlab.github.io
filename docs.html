Install, test, reproduce, evaluate.

QECTOR documentation is organized around the public package first: install from PyPI, verify the Python API, review benchmark artifacts, check claim boundaries, cite the DOI, and request commercial licensing when business use is involved.

Fast Start

Public PyPI path

pip install qector-decoder-v3
python -c "from qector_decoder_v3 import UnionFindDecoder, BlossomDecoder; print('QECTOR OK')"

This is the correct public command for normal users. Source builds are secondary and only for development, licensed source review, or unsupported environments.

Commercial pilot checklist

Evaluation path

  • Install qector-decoder-v3 from PyPI
  • Verify import with UnionFindDecoder and BlossomDecoder
  • Review GitHub README and benchmark artifacts
  • Compare QECTOR against PyMatching/Stim for your actual circuits
  • Document BP-OSD/LDPC needs, reproducibility requirements, and license scope
Request pilot
Claim hygiene.
Do not present QECTOR as faster than PyMatching, production SaaS, OEM-ready, or real-time hardware QEC infrastructure unless a specific reproducible artifact proves that exact claim.
