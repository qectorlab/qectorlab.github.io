QEC decoding for teams that need reproducible proof, not just a fast baseline.

QECTOR is a source-available Rust/Python QEC R&D platform that combines PyMatching-compatible validation, belief-matching accuracy experiments, BP-OSD for LDPC/qLDPC workflows, CPU/GPU batch paths, and artifact-hashed benchmark evidence.

Market position

Why QECTOR exists beside PyMatching and Stim

Not the fastest exact-MWPM pitch.
PyMatching remains the latency leader for exact MWPM on the tested workloads. QECTOR should not be sold as a universal PyMatching replacement.

The paid value

QECTOR is for labs, startups, and commercial R&D teams that need one legally clear platform for multi-decoder experiments, BP-OSD/LDPC workflows, GPU batch validation, reproducible artifacts, and benchmark reports.

Defensible moats

Five reasons QECTOR is not a me-too decoder

Proof

Reproducible by design

Environment snapshots, checked-in artifacts, SHA-256 proof paths, and reproducibility-first reports.

Accuracy mode

Belief-matching

LDPC/qLDPC

BP-OSD coverage

QECTOR Decoder

The core QEC decoder library for researchers, benchmarking workflows, and commercial QEC evaluation. Rust core with Python bindings via PyO3.

  • Robust AIO installer for Git Bash and Windows
  • Weighted MWPM workflows with PyMatching LER parity on tested workloads
  • Belief-matching accuracy mode with 33.7% lower LER at d=5 in the headline benchmark
  • BP-OSD for LDPC / qLDPC code experiments
  • CUDA and OpenCL batch decoding, bit-identical to CPU on tested configurations
  • 832 tests collected: 829 passed / 2 skipped / 1 expected xfailed

QECTOR Workbench

The planned product layer that turns QECTOR from a decoder library into a local reproducibility workstation for commercial R&D and lab validation.

  • Load .stim circuits and .dem files
  • Run QECTOR vs PyMatching comparisons
  • Generate PDF benchmark reports from real data
  • Export CSV / JSON artifacts with environment snapshots
  • Verify SHA-256 artifact hashes
  • Visualize LER, latency, threshold, memory, and scaling
