QEC decoding for teams that need reproducible proof, not just a fast baseline.

QECTOR is a source-available Rust/Python quantum error correction R&D platform for PyMatching-compatible validation, belief-matching accuracy experiments, BP-OSD for LDPC/qLDPC workflows, CPU/GPU batch paths, and artifact-backed benchmark evidence.

Real PowerShell install
git clone https://github.com/qectorlab/qector-decoder.git
cd qector-decoder
py -3.11 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip maturin
$env:PYO3_PYTHON = (Resolve-Path .\.venv\Scripts\python.exe).Path
.\.venv\Scripts\python.exe -m maturin develop --release --no-default-features

The live repository does not currently include install.py. Use the maturin build path shown here.

Market position

Why QECTOR exists beside PyMatching and Stim

Not the fastest exact-MWPM pitch.
PyMatching remains the latency leader for exact MWPM on the tested workloads. QECTOR should not be sold as a universal PyMatching replacement.

The paid value

QECTOR is for labs, startups, and commercial R&D teams that need one legally clear platform for multi-decoder experiments, BP-OSD/LDPC workflows, GPU batch validation, reproducible artifacts, and benchmark reports.

QECTOR Decoder

The core QEC decoder library for researchers, benchmarking workflows, and commercial QEC evaluation.

  • Rust/Python build from the canonical GitHub repository
  • Weighted MWPM workflows with PyMatching LER parity on tested workloads
  • Belief-matching accuracy mode: 34.8% lower observed LER at d=5 and 50.0% at d=7 in selected 3,000-shot correlated workloads
  • BP-OSD for LDPC / qLDPC code experiments
  • CUDA and OpenCL batch decoding with CPU bit-identity tests
  • Optional gRPC stack updated to tonic/prost 0.14
