Install QECTOR from the real GitHub repository.

The public repository does not currently ship install.py. Use the verified source-build path below: clone, create a virtual environment, install maturin, build the Rust/Python extension, then confirm the package import prints QECTOR OK.

PowerShell CommandGit Bash CommandOptional GPU Build
PowerShell
git clone https://github.com/qectorlab/qector-decoder.git
cd qector-decoder
py -3.11 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip maturin
$env:PYO3_PYTHON = (Resolve-Path .\.venv\Scripts\python.exe).Path
.\.venv\Scripts\python.exe -m maturin develop --release --no-default-features
.\.venv\Scripts\python.exe -c "from qector_decoder_v3 import UnionFindDecoder; print('QECTOR OK')"
Windows PowerShell

Official public install command

Run this from PowerShell. Rust must be installed first from rustup.rs. This command was verified on a second Windows PC from a fresh clone: the package built as qector-decoder-v3 0.5.0 and the final import check printed QECTOR OK.

git clone https://github.com/qectorlab/qector-decoder.git
cd qector-decoder

py -3.11 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip maturin

$env:PYO3_PYTHON = (Resolve-Path .\.venv\Scripts\python.exe).Path
.\.venv\Scripts\python.exe -m maturin develop --release --no-default-features

.\.venv\Scripts\python.exe -c "from qector_decoder_v3 import UnionFindDecoder; print('QECTOR OK')"
Git Bash

Git Bash equivalent

git clone https://github.com/qectorlab/qector-decoder.git
cd qector-decoder

python -m venv .venv
source .venv/Scripts/activate
python -m pip install --upgrade pip maturin

export PYO3_PYTHON="$(pwd -W)/.venv/Scripts/python.exe"
python -m maturin develop --release --no-default-features

python -c "from qector_decoder_v3 import UnionFindDecoder; print('QECTOR OK')"
Expected successful result

What success looks like

A successful Windows source build ends with maturin installing qector-decoder-v3-0.5.0, followed by the import check printing QECTOR OK. Messages such as Ignoring pytest: markers 'extra == "dev"' don't match your environment are normal for the base install because optional test and benchmark extras are not installed by the public CPU-safe command.

Finished `release` profile [optimized]
Built wheel for CPython 3.11
Installed qector-decoder-v3-0.5.0
QECTOR OK
Validation

Optional test suite

Run the full test suite only after the import check passes. These extras are for development validation, not the minimal public install.

.\.venv\Scripts\python.exe -m pip install "pytest>=7" "hypothesis>=6" "fastapi>=0.110" "uvicorn>=0.29" "httpx>=0.27"
.\.venv\Scripts\python.exe -m pytest python/tests -q --tb=short
Optional GPU build

Build GPU features only when the local machine is ready

The safe public command uses --no-default-features. CUDA/OpenCL builds are hardware/toolchain-specific and should be run separately.

# PowerShell, CUDA only
.\.venv\Scripts\python.exe -m maturin develop --release --no-default-features --features cuda

# PowerShell, OpenCL only
.\.venv\Scripts\python.exe -m maturin develop --release --no-default-features --features opencl
