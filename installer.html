Install QECTOR Decoder v3 from PyPI.

The normal public installation path is the PyPI wheel. Do not lead users to source-build commands unless they specifically need licensed source review or local development.

PyPI CommandVerify InstallOpen PyPI
PowerShell / terminal
pip install qector-decoder-v3
python -c "from qector_decoder_v3 import UnionFindDecoder, BlossomDecoder; print('QECTOR OK')"
Recommended command

Use the same command shown on PyPI.

This installs the official qector-decoder-v3 package from PyPI. On Windows, use the pip attached to the Python installation you actually run with python.

pip install qector-decoder-v3

If you maintain multiple Python installs, confirm them with pip --version, python --version, and py -0p. Avoid unsupported free-threaded Python ABIs until a matching wheel exists.

Verification

Expected successful result

python -c "from qector_decoder_v3 import UnionFindDecoder, BlossomDecoder; print('QECTOR OK')"

Expected output:

QECTOR OK
Supported public packages

Wheel-first public release.

Windows x64

Prebuilt CPython wheels are published for normal Windows users.

Linux x86_64

manylinux wheels are published for supported CPython versions.

macOS arm64

Apple Silicon wheels are published. Intel macOS may require a future wheel or licensed source-build path.

Source build boundary

Source builds are not the default user path.

Important: source builds are for local development, licensed source review, or special environments. They require a compatible Rust/Python toolchain and the appropriate source package. Public users should start with PyPI wheels.
