What it does

From circuit to reproducible report in one local session

QECTOR Workbench brings the decoder stack into a local fullstack application. Load your Stim circuits or Detector Error Models, run benchmark jobs across decoder backends, and export artifacts with verified hashes — all without sending data to a cloud service.

Load and parse

Load .stim circuit files and .dem Detector Error Model files directly. Workbench inspects the circuit structure, validates it, and prepares it for benchmark runs.

Run comparisons

Run QECTOR Blossom vs PyMatching head-to-head, belief-matching accuracy benchmarks, and BP-OSD experiments for qLDPC codes — all from the same interface.

Export artifacts

Export results as PDF reports, CSV tables, and JSON artifacts. Every artifact captures the environment snapshot and a SHA-256 hash for reproducibility verification.

Backend diagnostics

Inspect CPU, CUDA, and OpenCL backend status. Run calibration to find the actual CPU↔GPU crossover on your hardware before starting batch jobs.

Visualize results

Charts built from real artifact data — logical error rate curves, latency percentiles, threshold crossings, memory usage, and scaling behavior across distances.

Job management

FIFO job queue with cancel support. View live progress on running benchmark jobs. Error logs with full stack traces. Update notes and release history.

Who it's for

Researchers, developers, and commercial teams

QEC Researchers

Run reproducible benchmark suites without scripting infrastructure. Compare decoder performance on your own circuits with verified artifacts you can attach to papers.

Quantum Software Developers

Validate decoder integration before deployment. Compare QECTOR to PyMatching on your specific circuits and export evidence for technical review.

Commercial QEC Teams

Generate reproducible benchmark reports for due-diligence, investor validation, and internal documentation — without building and maintaining a custom benchmarking stack.

Enterprise Evaluation

Roadmap

Feature roadmap

Core benchmarking

01

Load .stim and .dem files

Parse Stim circuits and Detector Error Models with full validation and structure preview.

02

QECTOR vs PyMatching comparison

Head-to-head LER and latency benchmarks on the same circuit, same shots, same noise model.

03

Belief-matching benchmarks

Run belief-matching accuracy benchmarks against plain MWPM on correlated circuit-level noise.

04

BP-OSD / qLDPC experiments

Load bivariate-bicycle, hypergraph-product, and bicycle codes. Run BP-OSD and compare to ldpc reference.

Artifacts and reporting

05

PDF benchmark reports

Generate technical PDF reports with charts built from real artifact data — no fabricated figures.

06

CSV / JSON artifact export

Export all results as CSV tables and JSON artifacts with environment snapshot and SHA-256 hashes.

07

Environment and version capture

Every run stamps Python, Stim, PyMatching, QECTOR, git commit, CPU, and OS versions.

08

SHA-256 artifact verification

Verify that a benchmark artifact has not been modified since generation. Chain of custody for reproducibility.

Backend and diagnostics

09

CPU / CUDA / OpenCL diagnostics

Display backend status, available GPU devices, compute capability, and driver versions.

10

AutoDecoder calibration UI

Run calibration interactively to find the CPU↔GPU crossover on your hardware before batch runs.

11

GPU resilience monitoring

Display consecutive failures, total failures, degraded mode status, and recovery events in real time.

Visualization

12

Logical error rate curves

LER vs noise rate across distances. Threshold crossing visualization.

13

Latency percentiles

p50 / p90 / p95 / p99 latency charts. Hot-path vs cold-path split.

14

Memory and scaling

Peak memory usage and throughput scaling across distances and batch sizes.
